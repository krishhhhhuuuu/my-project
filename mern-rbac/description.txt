## ğŸ”¹ Objective

To design and implement a secure, full-stack web application that controls access to resources based on user roles and permissions, ensuring that unauthorized users cannot perform restricted actions.

## ğŸ”¹ Key Features

* User Authentication: Login using JSON Web Tokens (JWT).
* Role-Based Authorization: Admin, Editor, and Viewer roles with defined permissions.
* Admin Panel: Admins can view, add, and delete users.
* Ownership Control: Editors can only update their own content.
* Frontend UI Guards: React components and routes adapt based on user roles.
* Secure Passwords: Passwords are hashed using bcrypt.
* Validation: Input fields validated using `express-validator`.

## ğŸ”¹ System Workflow

1. User Login: A user logs in and receives a JWT token stored in localStorage.
2. Authorization Check: Each API route validates the token and checks user role using middleware.
3. Frontend Control: React uses `AuthProvider` and `PrivateRoute` to restrict access.
4. Admin Panel: Admins can manage users (add/delete) and assign roles.
5. Role Enforcement: Editors can update only their posts; Viewers can only read.
6. Logout: Clears session and redirects to login securely.
